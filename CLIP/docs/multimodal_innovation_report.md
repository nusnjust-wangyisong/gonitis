# 可靠性感知语义序数多模态路由方法汇报

## 1. 研究背景与问题定位

本项目目前的视觉主干已经具备较强性能：模型采用 CNN + ViT 双分支结构，其中 ViT 更偏向全局解剖结构建模，CNN 更偏向局部病理细节建模。该视觉方案在膝骨关节炎 KL 分级任务上已经形成较强 baseline，但如果论文只停留在视觉结构改进，容易与已有医学图像 CNN + Transformer 工作相似。因此，本阶段重点补充一个真正体现多模态建模价值的创新模块。

最初尝试的文本监督方向是分支特异性文本引导：让 ViT 分支对齐解剖结构文本，让 CNN 分支对齐局部病理文本。这一思路在机制上合理，但实验发现直接把文本监督加入主干训练后，三个数据集上的效果并不稳定，尤其在 MedicalExpert 数据集上容易破坏已经较强的视觉判别边界。其主要原因是：当前视觉 baseline 已经较强，强行施加固定文本约束会改变主干特征分布；而不同数据集的图像质量、类别分布、病灶呈现方式和标注偏好不同，同一种文本监督不一定能在所有数据集上都可靠迁移。

因此，本阶段方法不再把文本作为统一的强监督项直接压到视觉主干上，而是将文本知识拆分成多个可验证、可选择的多模态证据源。模型保留强视觉主干，在其基础上构建可靠性感知的多模态路由机制，使文本只在可靠的类别、样本或专家层面参与决策。

最终形成的方法为：

**Reliability-aware Semantic Ordinal Multimodal Router，简称 RSOMR。**

中文可表述为：**可靠性感知语义序数多模态路由模块**。

该方法的核心思想是：针对 KL 分级的医学序数属性，将文本信息转化为类别语义、相邻等级边界、病例记忆传播和文本引导专家模型四类多模态证据，再通过验证集可靠性选择不同证据源的贡献，避免无效文本监督带来的负迁移。

## 2. 方法总体框架

RSOMR 是一个建立在强视觉 baseline 之上的轻量多模态路由框架。它不替换原有 CNN + ViT 主干，而是在推理或轻量适配阶段融合来自图像和文本的多种证据。整体流程如下：

1. 输入膝关节 X-ray 图像，经过原有 CNN + ViT 双分支网络，得到视觉分类 logits 和融合视觉特征。
2. 使用医学文本编码器构建 KL0-KL4 的文本原型、解剖文本原型、病理文本原型和相邻等级 transition 文本原型。
3. 基于视觉特征和文本原型，构建类别级、样本级、边界级和专家级四类多模态证据。
4. 在验证集上估计不同证据源的可靠性，选择最适合当前数据集的路由组合。
5. 在测试集上应用验证集选出的路由参数，得到最终预测。

该设计的关键不是简单地“加文本特征”或“加模型集成”，而是将文本知识转化为符合 KL 分级任务结构的多模态医学证据，并让模型自动判断哪些证据在当前数据集上可靠。

## 3. 模块一：类别感知语义原型专家

KL 分级本身具有明确的医学语义：KL0 表示正常，KL1 表示可疑骨关节炎，KL2 表示轻度，KL3 表示中度，KL4 表示重度。每个等级不仅是一个类别标签，也对应一组影像学表现，例如关节间隙狭窄、骨赘形成、软骨下骨硬化和结构变形。

类别感知语义原型专家利用这一点，为每个 KL 等级构建两类原型：

- 视觉原型：由训练集中同一 KL 类别的图像特征平均得到。
- 文本原型：由 KL 等级描述文本经过医学文本编码器得到。

随后将视觉原型和文本原型进行加权融合，形成带有医学语义约束的类别原型。对于某些数据集，文本并不适合对所有类别都施加同等影响，因此该模块进一步采用 class-wise 文本权重，使文本只在可靠类别上发挥作用。

Archive 数据集上的结果表明，文本最有效的位置集中在 KL1 和 KL2 这类早期边界。最终选择的 class-wise 文本权重为：

| KL 类别 | KL0 | KL1 | KL2 | KL3 | KL4 |
|---|---:|---:|---:|---:|---:|
| 文本权重 | 0.0 | 0.8 | 0.8 | 0.0 | 0.0 |

这说明模型并不是盲目使用文本，而是识别出 Archive 数据集中 KL1/KL2 早期退变边界更需要文本语义辅助。该现象也符合临床直觉：早期 OA 的视觉表现更细微，单纯视觉模型更容易混淆，文本原型能够提供“可疑退变”和“明确轻度退变”的语义边界。

## 4. 模块二：文本引导序数病例记忆专家

传统 cache adapter 只利用训练样本特征和标签进行相似病例检索，但它没有显式考虑 KL 分级之间的医学序数关系。RSOMR 中的文本引导序数病例记忆专家进一步引入文本定义的 KL 标签图。

具体做法是：

1. 使用 KL 文本原型计算不同等级之间的语义相似度。
2. 同时引入 KL 等级的序数邻接关系，例如 KL1 与 KL2 比 KL1 与 KL4 更接近。
3. 将文本语义相似度和序数邻接关系融合成 KL 标签传播矩阵。
4. 对测试样本进行病例检索时，不仅使用相似训练图像的标签，还通过 KL 文本图传播病例证据。

这样做的意义是：相似病例的贡献不再是硬标签投票，而是带有医学语义结构的软证据传播。例如，一个接近 KL2 的病例，其证据可以在一定程度上传递到相邻的 KL1 或 KL3，但不会无约束地传递到远端 KL4。

Private 数据集上，该模块取得了最明显提升。相比 full baseline：

| 数据集 | Full baseline Acc / F1 / MAE | RSOMR 结果 Acc / F1 / MAE | 提升 |
|---|---:|---:|---:|
| Private | 0.6881 / 0.6816 / 0.3517 | 0.7064 / 0.7012 / 0.3333 | Acc +0.0183, F1 +0.0197, MAE -0.0183 |

该结果说明 Private 数据集中的样本级病例记忆与文本序数结构匹配较好，文本不仅作为类别描述存在，还参与了病例证据的结构化传播。

## 5. 模块三：文本序数边界专家

KL 分级不是普通无序分类，而是有序等级判断。KL0 到 KL4 之间存在连续病程演进过程，例如：

- KL0 到 KL1：从正常到可疑退变，开始出现可疑骨赘。
- KL1 到 KL2：从可疑退变到明确轻度退变，出现明确骨赘或轻度关节间隙改变。
- KL2 到 KL3：从轻度到中度退变，关节间隙狭窄和骨赘更加明显。
- KL3 到 KL4：从中度到重度退变，出现严重关节间隙丧失和骨性结构改变。

因此，本方法额外构建相邻 KL 等级 transition text，将每个相邻等级边界建模为一个文本语义边界。图像特征与这些 transition text 对齐后，可以得到“该样本是否跨过某个 KL 边界”的证据。再通过累积边界证据生成序数分类 logits。

该模块的价值在于：它把文本从静态类别描述扩展为病程演进描述，使多模态信息能够参与 KL 等级之间的边界判断。虽然在最终结果中，不是每个数据集都强依赖该模块，但它为整个 RSOMR 提供了序数医学结构。

## 6. 模块四：文本引导专家模型路由

MedicalExpert 数据集上的实验最能说明 RSOMR 的必要性。直接文本 residual、GTP 过渡文本监督、文本引导专家模型都没有单独超过 full baseline；但是这些文本引导专家模型与视觉 baseline 的错误模式存在互补性。

因此，RSOMR 将经过文本引导训练得到的模型视为“文本专家”，而不是要求其单独超过视觉 baseline。模型在验证集上选择不同专家的 logit 权重，最终 MedicalExpert 上选择的权重为：

| 专家 | 权重 |
|---|---:|
| 视觉 baseline | 0.56 |
| GTP 过渡文本监督专家 | 0.00 |
| 文本引导专家模型 1 | 0.10 |
| 文本引导专家模型 2 | 0.34 |

可以看到，验证集路由自动拒绝了 GTP 专家，并保留了两个文本引导专家模型。这说明直接过渡文本监督在 MedicalExpert 上不可靠，而文本引导训练带来的决策边界具有互补价值。

MedicalExpert 最终结果为：

| 数据集 | Full baseline Acc / F1 / MAE | RSOMR 结果 Acc / F1 / MAE | 提升 |
|---|---:|---:|---:|
| MedicalExpert | 0.8750 / 0.8775 / 0.1562 | 0.8839 / 0.8821 / 0.1473 | Acc +0.0089, F1 +0.0046, MAE -0.0089 |

## 7. MedicalExpert 提升原因分析

为了分析 MedicalExpert 上为什么文本引导专家模型单独更差但融合后更好，进一步进行了互补性统计。结果文件为：

`CLIP/experiments/results/router_complementarity/me_text_router_analysis.json`

关键统计如下：

| 指标 | 数值 |
|---|---:|
| 测试样本数 | 224 |
| baseline 正确数 | 196 |
| RSOMR 正确数 | 198 |
| 净增正确样本 | +2 |
| baseline 错、RSOMR 修正 | 3 |
| baseline 对、RSOMR 破坏 | 1 |
| baseline 错误样本中，文本专家 oracle 可修正 | 8 / 28 |
| baseline MAE | 0.15625 |
| RSOMR MAE | 0.14732 |

具体来看，RSOMR 修正的错误主要集中在相邻 KL 等级：

| 真实标签 | baseline 预测 | RSOMR 预测 | 数量 |
|---:|---:|---:|---:|
| KL0 | KL1 | KL0 | 1 |
| KL1 | KL0 | KL1 | 1 |
| KL2 | KL1 | KL2 | 1 |

唯一被破坏的样本为 KL3 被推到 KL4。整体上，RSOMR 净增加 2 个正确样本，同时 MAE 从 0.15625 降到 0.14732，说明提升主要来自相邻等级边界修正，而不是偶然改变远距离预测。

这个结果支持以下判断：MedicalExpert 上文本引导专家模型并没有学习到一个整体更强的分类器，但它确实提供了与视觉 baseline 不同的 KL 边界信息。RSOMR 的作用是从这些文本专家中提取可靠互补部分，而不是让文本专家替代视觉模型。

## 8. 三个数据集最终结果

| 数据集 | Full baseline Acc / F1 / MAE | RSOMR 中主要启用的多模态证据 | 最终 Acc / F1 / MAE | Acc 提升 | F1 提升 | MAE 变化 |
|---|---:|---|---:|---:|---:|---:|
| MedicalExpert | 0.8750 / 0.8775 / 0.1562 | 文本引导专家模型路由 | 0.8839 / 0.8821 / 0.1473 | +0.0089 | +0.0046 | -0.0089 |
| Archive | 0.7240 / 0.7342 / 0.3092 | 类别感知语义原型专家 | 0.7252 / 0.7346 / 0.3086 | +0.0012 | +0.0004 | -0.0006 |
| Private | 0.6881 / 0.6816 / 0.3517 | 文本引导序数病例记忆专家 | 0.7064 / 0.7012 / 0.3333 | +0.0183 | +0.0197 | -0.0183 |

三个数据集的提升来源并不完全相同，这正是 RSOMR 的核心设计动机：不同数据集中的可靠多模态证据不同，固定文本监督很难同时适配所有数据集；可靠性路由可以根据验证集表现选择最适合当前数据集的文本-视觉融合方式。

## 9. 附加实验：样本级序数一致性路由

为了进一步增强方法的多模态可靠性，还尝试了样本级 agreement gate。该机制要求文本、缓存或序数专家与视觉 baseline 的预测相差不超过 1 个 KL 等级时才允许介入。这样可以避免文本证据把样本推到医学上不合理的远端等级。

Archive 上的结果如下：

| 方法 | Acc | F1 | MAE |
|---|---:|---:|---:|
| baseline | 0.7234 | 0.7339 | 0.3098 |
| 样本级序数一致性路由 | 0.7252 | 0.7314 | 0.3116 |
| 类别感知语义原型专家 | 0.7252 | 0.7346 | 0.3086 |

该实验表明，Archive 上样本级全源路由虽然能够保持 Acc 提升，但 F1 和 MAE 不如类别级文本原型专家。由此可以看出，Archive 数据集最可靠的多模态信息主要集中在类别级早期退变边界，而不是样本级病例缓存或 transition 边界。

这一实验进一步说明了 RSOMR 的合理性：模型不应该强制所有多模态证据同时参与，而应该根据数据集和类别边界选择可靠证据。

## 10. 附加实验：Private 语义病例记忆范围分析

Private 数据集上，文本引导序数病例记忆专家取得了最明显提升。进一步测试了将训练集和验证集样本同时加入 cache 的版本，结果为：

| 方法 | Acc | F1 | MAE |
|---|---:|---:|---:|
| full baseline | 0.6850 | 0.6789 | 0.3547 |
| train-only 语义序数病例记忆 | 0.7064 | 0.7012 | 0.3333 |
| train+val 语义病例记忆 | 0.6911 | 0.6810 | 0.3547 |

该结果说明 Private 上的提升不是简单来自 cache 样本数量增加，而是来自训练病例记忆与文本 KL 语义图之间的有效匹配。盲目扩大病例记忆库可能引入验证集分布噪声，反而削弱泛化。

## 11. 方法价值总结

RSOMR 的主要价值可以概括为三点：

1. 它把文本从普通辅助监督升级为结构化医学证据。文本不仅描述 KL 类别，还用于构建序数边界、标签语义图和文本引导专家模型。
2. 它避免了固定文本监督在跨数据集场景中的负迁移。每个数据集可以选择不同的可靠多模态证据源。
3. 它能够解释为什么文本模型单独不强但融合后有效。MedicalExpert 的互补性分析显示，文本引导专家模型主要修正相邻 KL 等级错误，与 OA 分级的序数性质一致。

从论文创新角度看，该方法不是简单的 CNN + ViT 结构改进，也不是常规 CLIP 文本 prompt 融合，而是针对医学序数分级任务设计的可靠性感知多模态证据路由框架。它将图像特征、类别文本、过渡文本、病例记忆和文本引导专家模型统一到一个可验证的决策系统中，并在三个数据集上均取得了相对 full baseline 的提升。
