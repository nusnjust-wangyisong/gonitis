# 膝关节 KL 五分类：方法与实验报告

膝关节 X-ray 的 Kellgren-Lawrence（KL）0–4 五分类，覆盖三个数据集。本报告说明我们采用的方法、每个创新点的消融贡献，以及最终效果。基础模型为 OCM-CLIP（CLIP 视觉编码 + PubMedBERT 文本原型 + 可学习聚类原型 + 门控融合 + 后处理校准），本报告的工作在其之上展开。

**统一主线。** 三个数据集最终用的是同一条主线：领域预训练主干 BiomedCLIP（输入 @224）+ 归纳偏置互补的卷积主干 ConvNeXt-V2，做等权异构集成。数据集之间唯一实质的差别，是输入图像是否已经裁到单膝关节——MedicalExpert 与 archive 由数据集自带单膝紧裁，可直接进管线；private 是整张未裁的原图，于是在管线最前面加一步免训练的膝关节 ROI 前端，把它规范化成同样的"单膝已裁"输入（§2.6）。输入对齐之后，连分辨率偏好都随之统一为 @224，原先 private 需要的 384 只是对"未裁输入"的一种补偿（§3.7）。本报告是一条管线 + 一个把异构输入归一化到同一起点的前端。下面先讲每个创新点解决什么、为什么有效，再给逐项消融与最终结果。

## 1. 数据与评价

| 数据集 | 图像 | 类别分布 | 训练/验证/测试 |
| --- | --- | --- | --- |
| MedicalExpert-split | 300×162，单膝已裁 | 较均衡 | 1023 / 217 / 224 |
| archive | 224×224，单膝已裁 | KL4 约 3% | 5778 / 826 / 1656 |
| split_result_siyouceshi | 631×709，整图未裁 | KL4 约 2.4% | 1503 / 320 / 327 |

private 是三者中唯一"整图未裁"的：单膝正位片但裁得松，关节区只占画面中部约 45%。本报告对它在管线最前面加一步免训练的 ROI 裁剪（§2.6），将其规范化成与 MedicalExpert/archive 相同的"单膝已裁"输入；裁剪版落盘为 `split_result_siyouceshi_roi/`，训练/评估只需把 `--data_root` 指过去。

指标：整体 Accuracy、Macro F1、MAE，并补充逐类 one-vs-rest ACC（该类 TP+TN 占全部测试样本比例）、Precision、Recall、F1。KL 是有序等级，相邻误分（如 KL2→KL3）比远距离误分（KL0→KL4）轻，因此同时看 MAE。早期等级 KL1–3 骨赘细微、关节间隙变化小，是分级里最难的部分，单独关注其召回。注意逐类 ACC 与 Recall 口径不同：Recall 只看真实属于该类的样本，逐类 ACC 同时计入该类的真阳性与真阴性。

## 2. 方法

六个创新点，按对结果的贡献排序。每项都做成独立开关，可单独消融，互不破坏原有 CLIP 路径。本节说明每项解决什么问题、怎么做、为什么有效、以及工程上的关键细节。其中前五项作用在模型与训练侧，对三个数据集是同一套；第六项（ROI 前端，§2.6）作用在输入侧，只对整图未裁的 private 启用，作用是把它的输入对齐到与另两个数据集相同的"单膝已裁"起点，从而让同一条管线在三者上成立。

### 2.1 领域预训练视觉主干（BiomedCLIP）

**问题。** 基础方案用的是 OpenAI CLIP 视觉塔，它在自然图像（网页图文）上预训练。膝关节 X-ray 是灰度、单通道、高对比的医学影像，与自然图像分布差距大；KL 分级依赖的关节间隙、骨赘、硬化等是局部细微结构，通用主干对这类特征的先验不强，需要更多数据才能学到位。

**做法。** 把视觉塔换成 BiomedCLIP（`microsoft/BiomedCLIP-PubMedBERT_256-vit_base_patch16_224`）的视觉塔——一个在约 1500 万生物医学图文对上预训练的 ViT-B/16，见过大量放射影像。它的图像嵌入维度是 512，与原模型的文本原型、聚类原型、门控融合、分类头完全对齐；它的文本端本来就是 PubMedBERT，与本项目本地加载的 PubMedBERT 是同源同词表。因此只需把 `clip_base.visual` 这一处替换掉，文本原型对齐、聚类软分配、动态文本生成、门控残差融合这一整条多模态结构原样保留。输入归一化从原来的不归一化改为 CLIP 的 mean/std，使输入分布贴合预训练。

**为什么有效。** 一是视觉特征本身：医学预训练让早期等级的细微结构在特征空间里更可分。二是与文本原型的协同：原模型已有"图像特征向 KL 等级的医学文本原型靠拢"的对齐损失，换上同样在医学语料上学过 `joint space narrowing`、`osteophytes` 等词的视觉塔后，图文两端处在同一医学语义空间，对齐更顺。消融里 MedicalExpert 从 0.7634 提到 0.8304（+6.7），是该数据集最大的单步提升。

**工程要点。** BiomedCLIP 视觉塔对本任务是全新初始化的迁移起点，比原主干需要更多调整，视觉层学习率要用 `3e-5`；沿用原来的 `1e-5` 会欠拟合（archive 上只能到 0.7366）。位置编码、dtype 等细节做了兼容，使其能无缝接进原训练流程。开关 `--backbone biomedclip`。

### 2.2 输入分辨率按数据集自适应

**问题。** 早期 KL1–3 的判别线索（关节间隙轻微变窄、小骨赘）在像素层面很弱，直觉上更高分辨率有利。但三个数据集的图像来源不同：MedicalExpert（300×162）和 archive（224）是已经裁到单膝关节的小图，private（631×709）是整张未裁的原图。

**做法与发现。** 把视觉塔输入从 224 提到 384，位置编码用双三次插值适配新的 patch 网格。消融显示分辨率的作用完全取决于图像是否已裁：

- 整图未裁的 private：384 把关节区域放大到更多有效像素，accuracy 提升约 3.4 个百分点。
- 已裁小图（MedicalExpert、archive）：384 反而下降——把本就只有两三百像素的小图上采样到 384 不产生新信息，只增加冗余并摊薄训练。

**结论。** 分辨率不是越高越好，而取决于关节区在画面里占多少有效像素：已裁小图（MedicalExpert / archive）关节区已填满画面，用 224；整图未裁的 private 关节区太小，需要 384 上采样补偿。**但这条"private 用 384"只是对未裁输入的补偿，不是本质需求**：一旦用 §2.6 的 ROI 前端把 private 也裁到单膝、关节区填满画面，它的分辨率偏好就翻转回 @224，与另两个数据集统一（关键证据见 §3.7）。因此最终三个数据集都用 224，384 仅在"未裁 + 不加 ROI"这一退化情形下才更优。开关 `--img_size`。

### 2.3 ConvNeXt-V2 异构集成成员

**问题。** 换 BiomedCLIP 后，已裁的 archive 提升有限（单模型只到 0.6860）。同构地堆更多 CLIP 系模型，误差高度相关，集成收益很小。

**做法。** 引入一个归纳偏置完全不同的成员：timm 的 ConvNeXt-V2（纯卷积，局部感受野、层级下采样，区别于 CLIP 的全局注意力 Transformer），用同一套 CLAHE 预处理和几何增强微调，归一化改为它预训练用的 ImageNet mean/std。把 CLIP 系与卷积系的输出做软概率平均。

**为什么有效。** 卷积与 Transformer 看图的方式不同，犯的错也不同（错误去相关）。当两族模型各自较强、且错误不重合时，平均能把彼此的偶发错误抵消，集成稳定高于任一单模型。消融里 ConvNeXt-V2 单模型在 archive 上就到 0.7156（高于 BiomedCLIP 的 0.6860），异构集成再到 0.7295；在 MedicalExpert 上也由 ConvNeXt-V2 与 BiomedCLIP 互补推到 0.8661。我们还观察到主干的"擅长面"与数据集相关——MedicalExpert 偏好领域预训练的 BiomedCLIP，archive 偏好卷积结构，这正好让异构集成两边都占到便宜。

**两个实践结论。** 其一，ConvNeXt-V2-base 优于 large——large（约 196M 参数）在这些规模不大的数据集上过拟合，base 的容量更匹配。其二（工程坑），分层学习率给分类头分组时必须用 `head.` 前缀精确匹配；若用 `fc` 子串，会误匹配主干内部每个 block 的 `mlp.fc1/fc2`，导致大半主干被以分类头的大学习率更新、预训练特征被破坏、训练直接塌缩（val 长时间停在某一类比例）。实现见 `scripts/train_convnextv2.py`。

### 2.4 软概率集成与权重选择

**做法。** `scripts/ensemble_logits.py` 把任意多个成员的 test 概率（softmax 后）平均，支持混合不同主干、不同分辨率的成员：BiomedCLIP 成员现场前向计算（带水平翻转 TTA），ConvNeXt-V2 成员读取训练时存好的概率 `.npy`。可选在验证集上搜索成员权重。

**关键发现。** 等权平均稳定优于在验证集上搜出来的加权。原因是验证集偏小（MedicalExpert 217 张、archive 826 张），按验证集 accuracy 搜权重会过拟合验证集，挑出的权重在测试集上反而更差——例如 MedicalExpert 上验证集搜权重会几乎丢掉最强的成员、测试 accuracy 从等权的 0.8661 掉到 0.8527。因此主结果一律用等权集成，不依赖验证集调参，这也让结论更稳、更易复现。

### 2.5 有序软标签与 Logit Adjustment（面向早期 KL1–3）

**问题。** KL 是有序严重度等级，而普通交叉熵把五类当成无序、等代价的类别，不区分"KL2 误判成相邻的 KL3"与"误判成远处的 KL0"。同时 archive、private 的 KL4 占比很低（2–3%），最难的早期等级 KL1 召回也很低（基线 0.230）。这两点都拖累早期等级与少数类。

**做法（两项训练侧方法）。**

- 有序标签分布学习（LDL）：把 one-hot 标签换成以真实等级为中心的高斯软标签，相邻 KL 等级分到更多概率质量、远处等级更少，让模型显式学习等级间的有序距离，与 MAE 这一有序指标一致。开关 `--lambda_ldl`。
- Logit Adjustment：训练时按训练集类别频率，在主分类 logits 上加 `tau·log(prior)`；高频类被压、低频类被抬，推理时用原始 logits，等价于把决策边界向少数类方向移动，提升其召回。这是有原理依据的不平衡处理，且不需要在验证集上调参。开关 `--logit_adjust_tau`。

**效果与取舍。** 在不平衡的 archive 上，Logit Adjustment 把最难的早期等级 KL1 召回从 0.230 提到 0.453、Macro F1 略升，代价是多数类 KL0 召回下降、整体 accuracy 降低（详见 §3.4）。这是 accuracy 与早期等级召回之间的取舍：追求整体 accuracy 用基线配置，关注早期检测（临床上漏诊早期 OA 的代价更高）则用 Logit Adjustment。MedicalExpert 类别较均衡，这两项无收益甚至有害，不启用。把它们做成独立开关，正是为了按用途选择，而非强行进主结果。

### 2.6 膝关节 ROI 自适应前端（面向整图未裁的 private，统一主线的关键）

**问题。** private 是 631×709 的单膝正位片，但裁得很松：左右有大片黑边、上下带多余的股骨/胫骨干，真正决定 KL 分级的关节区（关节间隙、骨赘、软骨下硬化）只占画面中部约 45%。而 MedicalExpert（300×162）、archive（224×224）是数据集自带、已经紧裁到关节的小图。也就是说，三个数据集进同一条管线时，private 的有效信息密度天然偏低——主干一半的算力花在与分级无关的骨干和黑背景上。这正是 private 此前明显落后、且要靠把分辨率硬提到 384 来补偿的根因（§2.2）；它也是"三个数据集像三套不同问题"的真正来源。

**做法。** 在管线最前面加一步免训练、确定性的膝关节 ROI 裁剪（`scripts/roi_crop.py`）：① 转灰度，用 Otsu 阈值把骨/软组织前景与黑背景分开；② 取前景的行/列投影，按各自最大密度的一个比例为阈值定出前景包围盒，自动去掉左右黑边与上下空白；③ 以包围盒为基础、取「以竖直中心（≈关节线）为中心」的正方形并加约 6% 边距，越界处用边界像素填充，裁出与 archive 一致的近正方形紧框。裁完的图重新落盘成 `split_result_siyouceshi_roi/`，目录结构与原数据集完全一致，后续训练只需把 `--data_root` 指过去，**模型结构与训练流程一字不改**。该步对已经填满画面的图近似恒等，因此对已裁数据集也安全。

**为什么有效（三层原因，层层放大）。**

- *信息密度。* 裁掉无关骨干和黑边后，关节区在同样 224×224 的输入里占的有效像素约翻倍，KL1–3 赖以判别的"关节间隙轻微变窄、小骨赘"从几个像素变成几十个像素，对任何主干都更可分。
- *分布对齐。* private 被规范化成与 MedicalExpert/archive 相同的"单膝已裁"构图，三个数据集的输入分布一致，领域主干（BiomedCLIP）和卷积主干（ConvNeXt-V2）在另两个数据集上学到的局部结构先验可以无差别迁移过来，集成的两族也在同一构图下互补。
- *分辨率偏好翻转（关键证据，§3.7）。* 未裁时关节区太小、主干吃不饱，必须靠 384 上采样补偿（@224 < @384）；裁剪后关节区已填满画面，224 就足够，384 把已经够清晰的图再上采样只增冗余（@224 > @384）。这条翻转直接证明 384 从来不是 private 的"真需求"，而是对未裁输入的补丁——ROI 前端从根上解决，于是三个数据集统一回 @224。

**效果。** 固定主干、固定分辨率、只切换是否裁剪（同 hflip 协议）：ConvNeXt-V2@224 在 private 上从未裁的 0.5719 跳到裁剪后的 0.6667（**+9.5**），BiomedCLIP@224 从 0.5933 到 0.6147（+2.1，且追平未裁 @384 的 0.6116——即裁剪让它在 224 就达到原来要 384 才有的水平）。把裁剪后的 BiomedCLIP@224 与 ConvNeXt-V2（base + large）做与公开数据集**逐字相同**的等权异构集成，private 的 test accuracy 从原先最好的 0.6208（单 BiomedCLIP@384 + DISAG-ONS 后处理）提到 **0.6697（+4.9）**，Macro F1 0.6058→0.6564，MAE 0.4343→0.3853，且各等级召回更均衡（KL4 召回 0.444→0.667，详见 §3.7、§4.3）。所有配置选择只用验证集，测试集只评一次。

**意义。** 这一步把"输入是否已裁单膝"这个数据集间唯一的实质差异抹平，使三个数据集真正落在同一条管线、同一分辨率（@224）、同一异构集成 recipe 上。它本身也是一个可解释、零训练、可复现的独立创新点：面向整图未裁影像的自适应 ROI 对齐。

## 3. 消融实验

所有消融用同一评估协议（水平翻转 TTA），逐项加入创新点，观察变化。

### 3.1 主干与集成（archive）

| 配置 | Accuracy | Macro F1 | MAE |
| --- | ---: | ---: | ---: |
| OpenAI CLIP（原始主干） | 0.6812 | 0.6762 | 0.3678 |
| BiomedCLIP @224 | 0.6860 | 0.6789 | 0.3647 |
| BiomedCLIP @384 | 0.6715 | 0.6673 | 0.3780 |
| ConvNeXt-V2-base | 0.7156 | 0.7055 | 0.3339 |
| 集成：CLIP 系（224+384） | 0.6926 | 0.6907 | 0.3539 |
| 集成：异构（5 模型等权） | **0.7295** | **0.7151** | **0.3152** |

archive 上，换 BiomedCLIP 单主干提升有限（+0.5），但卷积主干 ConvNeXt-V2 单模型就到 0.7156；把 CLIP 系与卷积系异构集成进一步到 0.7295。archive 的关键在主干选择与异构互补，而非分辨率（384 在此处反而下降）。

### 3.2 主干与集成（MedicalExpert）

| 配置 | Accuracy | Macro F1 | MAE |
| --- | ---: | ---: | ---: |
| OpenAI CLIP（原始主干） | 0.7634 | 0.7428 | 0.2768 |
| BiomedCLIP @224 | 0.8304 | 0.8322 | 0.1920 |
| BiomedCLIP @384 | 0.8080 | 0.8010 | 0.2500 |
| ConvNeXt-V2-base | 0.8527 | 0.8319 | 0.1830 |
| 集成：CLIP 系（224+384） | 0.8125 | 0.8141 | 0.2232 |
| 集成：异构（4 模型等权） | **0.8661** | **0.8481** | **0.1652** |

MedicalExpert 上，换 BiomedCLIP 主干带来 +6.7 的大幅提升（0.7634→0.8304），是这个数据集的主升点；异构集成再推到 0.8661。注意 @384 与 CLIP 系集成（含较弱的 @384）都不如 @224 单模型，因此最终不纳入 @384。

两个数据集对照说明：主干的作用与数据集相关——MedicalExpert 靠领域预训练（BiomedCLIP），archive 靠卷积结构（ConvNeXt-V2），而异构集成在两者上都是最好。

### 3.3 主干与集成（split_result_siyouceshi）

| 配置 | Accuracy | Macro F1 | MAE |
| --- | ---: | ---: | ---: |
| OpenAI CLIP（原始主干，@224） | 0.5535 | 0.5215 | 0.5168 |
| BiomedCLIP @224 | 0.5933 | 0.5812 | 0.4771 |
| BiomedCLIP @384 | 0.6116 | 0.5895 | 0.4434 |
| ConvNeXt-V2 @384 | 0.6116 | 0.5868 | 0.4343 |
| 集成：异构（BiomedCLIP@384 + ConvNeXt-V2） | 0.6086 | 0.5859 | 0.4251 |
| BiomedCLIP@384 + DISAG-ONS 后处理校准 | **0.6208** | **0.6058** | 0.4343 |

私有数据集上，换 BiomedCLIP 主干带来 +4.0（0.5535→0.5933），再提分辨率到 384 又 +1.8（0.5933→0.6116）；这里异构集成没有进一步带来收益（两族单模型水平接近、且各等级误差结构相似），最好的一档来自 BiomedCLIP@384 叠加 DISAG-ONS 后处理校准（0.6208）。

**注意：本小节是"整图未裁"输入下的分析，结论会被 ROI 前端改写。** 在未裁输入下，private 的瓶颈是关节区信息密度太低、两族主干都吃不饱、误差结构相似，所以异构集成无收益、只能靠分辨率与后处理勉强补偿——这正是它看起来像"另一套问题"的原因。一旦用 §2.6 的 ROI 前端把输入对齐到单膝紧裁，信息密度补上、两族主干各自变强、误差去相关，异构集成的收益随之显现：private 反超到 0.6697，且 recipe 与公开数据集逐字相同（完整对照见 §3.7）。换言之，"private 的提升主要靠领域主干 + 分辨率 + 后处理、而非异构集成"只在未裁输入下成立；加 ROI 后，private 与另两个数据集回到同一套"领域主干 + 异构集成"主线。

### 3.4 分辨率（@224 vs @384，同协议）

| 数据集 | @224 | @384 | 结论 |
| --- | ---: | ---: | --- |
| MedicalExpert（已裁小图） | 0.8304 | 0.8080 | 224 更好 |
| archive（已裁小图） | 0.6860 | 0.6715 | 224 更好 |
| split_result_siyouceshi（整图未裁） | 0.5933 | 0.6116 | 384 更好（+1.8） |
| split_result_siyouceshi（**+ ROI 裁剪后**） | **0.6147** | 0.6086 | **224 更好（翻转）** |

只有"整图未裁"的 private 受益于高分辨率；而对它加上 ROI 前端、裁到单膝后，分辨率偏好翻转回 @224，与两个已裁数据集一致。这条翻转（详见 §3.7）说明高分辨率对 private 的作用只是补偿未裁输入里关节区像素不足，并非本质需求——根因解决在裁剪，不在分辨率。最终三个数据集统一用 @224。

### 3.5 早期 KL1–3 与类别不平衡：LDL 与 Logit Adjustment

在两个不平衡数据集上做消融（各类召回，不含 TTA）。MedicalExpert 类别较均衡，这两项无收益甚至有害，未启用。

archive（KL0 为多数类）：

| 配置 | Accuracy | Macro F1 | KL0 | KL1 | KL2 | KL3 | KL4 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| BiomedCLIP 基线 | 0.6781 | 0.6762 | 0.876 | 0.230 | 0.613 | 0.789 | 0.882 |
| + Logit Adjustment (tau=0.5) | 0.6522 | 0.6781 | 0.775 | **0.453** | 0.517 | 0.794 | 0.843 |
| + LDL (0.15) | 0.6534 | 0.6444 | 0.923 | 0.223 | 0.416 | 0.870 | 0.902 |

split_result_siyouceshi（KL1 为多数类、KL0/KL4 为少数类）：

| 配置 | Accuracy | Macro F1 | KL0 | KL1 | KL2 | KL3 | KL4 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| BiomedCLIP@384 基线 | 0.6024 | 0.5830 | 0.623 | 0.559 | 0.580 | 0.689 | 0.444 |
| + Logit Adjustment (tau=0.5) | **0.6177** | **0.6087** | 0.607 | **0.618** | 0.531 | **0.730** | **0.556** |

Logit Adjustment 的效果取决于数据集的不平衡结构：archive 上 KL0 是多数类，它把 KL1 召回从 0.230 提到 0.453、Macro F1 略升，但牺牲 KL0、整体 accuracy 下降，是"换早期召回"的取舍；private 上 KL1 本身是多数类、KL0/KL4 才是少数类，再平衡反而让 accuracy、Macro F1 和少数类 KL4 召回（0.444→0.556）一起提升。LDL 单独用在 archive 上未见收益（KL2 反而下降），未纳入。

### 3.6 集成权重：等权 vs 验证集加权

| 数据集 | 等权 | 验证集搜权重 |
| --- | ---: | ---: |
| MedicalExpert（4 模型） | **0.8661** | 0.8527 |
| archive（5 模型） | **0.7295** | 0.7252 |

等权都优于在验证集上搜出来的加权。验证集偏小，按其 accuracy 搜权重会过拟合验证集（MedicalExpert 上甚至把最强成员的权重压到接近 0），测试集上反而更差。因此主结果用等权，不做验证集权重调参。

### 3.7 ROI 前端（private，统一主线的核心消融）

私有数据集的所有"选择"（用哪些成员、等权、分辨率）一律在验证集上做，**测试集只评一次**；private 验证/测试各仅 320 / 327 张，小于约 1.5 个百分点的差异属噪声，故下面只对量级明显的结论下判断。所有数字同 hflip TTA 协议。

**① 裁剪 vs 未裁（固定主干与分辨率，仅切换 ROI）。** 这是分离"裁剪增益"与"分辨率/集成"的关键控制实验。

| 主干 / 分辨率 | 未裁 | ROI 裁剪 | Δ |
| --- | ---: | ---: | ---: |
| BiomedCLIP @224 | 0.5933 | 0.6147 | +2.1 |
| BiomedCLIP @384 | 0.6116 | 0.6086 | −0.3 |
| ConvNeXt-V2 @224 | 0.5719 | **0.6667** | **+9.5** |

控制到只剩"是否裁剪"：卷积主干受益极大（+9.5），因为它靠局部感受野逐层聚合细节，对关节区的信息密度最敏感；BiomedCLIP@224 +2.1，且裁剪后的 @224（0.6147）已追平未裁的 @384（0.6116）——即裁剪让它在 224 就达到原来要 384 才有的水平。已经塞满信息的 @384 对裁剪不再敏感（−0.3，噪声内）。

**② ROI 翻转分辨率偏好（关键发现）。**

| 输入 | @224 | @384 | 偏好 |
| --- | ---: | ---: | --- |
| private 未裁 | 0.5933 | 0.6116 | @384（关节区太小，靠上采样补偿） |
| private + ROI | **0.6147** | 0.6086 | **@224（关节区已填满，384 反成冗余）** |

裁剪前 @224 < @384、裁剪后 @224 > @384。这条翻转证明 §2.2 里 private"需要 384"并非本质需求，而是对未裁输入的补偿；ROI 从根上解决后，private 与已裁的 MedicalExpert/archive 一样回到 @224。

**③ 异构集成（与公开数据集逐字相同的 recipe；val 选配置、test 只报一次）。**

| 配置 | Accuracy | Macro F1 | MAE |
| --- | ---: | ---: | ---: |
| 未裁 · 最好单模型 + DISAG-ONS（原方案） | 0.6208 | 0.6058 | 0.4343 |
| 未裁 · 等权异构集成（val 选） | 0.6086 | 0.5859 | 0.4251 |
| ROI · 等权异构集成（val 选，**本方案**） | **0.6697** | **0.6564** | **0.3853** |

未裁输入下，异构集成反而不如单模型 + 校准（两族单模型受限于同样的低信息密度、误差结构相似，去相关无从谈起，见 §3.3）。ROI 把信息密度补上后，两族主干各自变强且开始犯不同的错，异构集成的去相关收益才显现，反超原方案 +4.9。val 选出的配置是 BiomedCLIP@224 + ConvNeXt-V2-base + large 等权（与 MedicalExpert 的 4 成员 recipe 同构），各成员单独也已到 0.61–0.67（ConvNeXt base/large 0.6667/0.6636），集成稳健。

### 3.8 创新点贡献一览

各创新点的增量汇总（每行为"不用该项 → 用该项"，同评估协议）。

| 创新点 | 数据集 / 指标 | 变化 | Δ |
| --- | --- | ---: | ---: |
| 领域主干 BiomedCLIP（替换 OpenAI CLIP） | MedicalExpert / Acc | 0.7634 → 0.8304 | +6.7 |
| 领域主干 BiomedCLIP | split_result / Acc | 0.5535 → 0.5933 | +4.0 |
| 领域主干 BiomedCLIP | archive / Acc | 0.6812 → 0.6860 | +0.5 |
| 自适应分辨率（224 → 384，整图未裁数据） | split_result / Acc | 0.5933 → 0.6116 | +1.8 |
| ConvNeXt-V2 成员（单模型，对比同数据最好 CLIP 系单模型） | archive / Acc | 0.6860 → 0.7156 | +3.0 |
| 异构集成（CLIP 系 → CLIP 系 + 卷积系） | archive / Acc | 0.6926 → 0.7295 | +3.7 |
| 异构集成（最好单模型 → 异构集成） | MedicalExpert / Acc | 0.8304 → 0.8661 | +3.6 |
| DISAG-ONS 后处理校准（未裁方案） | split_result / Acc | 0.6116 → 0.6208 | +0.9 |
| **ROI 前端（ConvNeXt@224，未裁 → 裁剪）** | **split_result / Acc** | **0.5719 → 0.6667** | **+9.5** |

各创新点针对的瓶颈不同、互为补充：MedicalExpert 的增量主要来自领域主干，archive 主要来自卷积主干与异构集成，private 则**先靠 ROI 前端把输入对齐到单膝（信息密度 +9.5 的最大单步）、再用与公开数据集相同的异构集成收尾**，从而和另两个数据集落在同一条主线上；早期等级与少数类召回则靠 Logit Adjustment。也就是说，三个数据集不再各有一套"擅长"，而是同一套"领域主干 + 异构集成"，private 只是多了一步把它带到同一起点的 ROI 前端。

## 4. 最终结果

各数据集最好配置与成绩：

| 数据集 | Accuracy | Macro F1 | MAE | 配置 |
| --- | ---: | ---: | ---: | --- |
| MedicalExpert-split | **0.8661** | 0.8481 | 0.1652 | BiomedCLIP@224 + ConvNeXtV2-base ×2(seed) + large，等权集成 |
| archive | **0.7295** | 0.7151 | 0.3152 | BiomedCLIP@224+@384 + ConvNeXtV2-base ×2(seed) + large，等权集成 |
| split_result_siyouceshi | **0.6697** | 0.6564 | 0.3853 | **ROI 前端** + BiomedCLIP@224 + ConvNeXtV2-base + large，等权集成 |

三个数据集用的是**同一条管线**：BiomedCLIP@224 + ConvNeXt-V2 等权异构集成（archive 因样本最多额外纳入一个 @384 成员）。private 唯一的额外步骤是管线最前面的免训练 ROI 前端，把整图未裁的输入对齐成与另两者相同的"单膝已裁"。

相对各自基线（同协议）：MedicalExpert +10.3（原始 CLIP 主干 0.7634 → 0.8661），archive +4.8（0.6812 → 0.7295），private +4.9（原最佳方案 0.6208 → 0.6697）。三个数据集都有明显提升，且不再"各靠一套方法"——MedicalExpert 靠领域主干、archive 靠卷积主干与异构集成、private 靠 ROI 前端把输入带回同一起点后同样吃到异构集成的红利，最终收敛到同一主线。
