# 面向膝关节 Kellgren–Lawrence 分级的结构化文本–视觉多模态序数证据框架

> 本稿提出三项核心创新：①全局–局部跨注意力双分支融合、②多尺度局部建模与分层热启动优化（前两项构成单模型视觉强基线），③RSOMR 可靠性感知语义序数多模态证据路由（文本–视觉多模态创新），构成一个面向 KL 序数分级的可验证多模态框架。

---

## 摘要

膝关节骨关节炎的 Kellgren–Lawrence（KL）分级是一项具有明确临床序数语义的细粒度分级任务：KL0–KL4 对应从正常、可疑、轻度到中重度的连续病程演进。现有方法多将其视为扁平的五分类，既未利用分级背后的医学文本语义，也未显式建模等级之间的序数结构。本文提出一个**结构化文本–视觉多模态序数框架**，包含三项核心创新：（1）**全局–局部跨注意力双分支编码**，在单一模型内端到端融合 Transformer 的全局解剖建模与卷积的局部骨赘细节；（2）**多尺度局部建模与分层热启动优化**，自适应融合 stage-2/3 局部特征并稳定双分支训练；（3）**RSOMR**——可靠性感知语义序数多模态证据路由，将 KL 类别文本原型、序数病例记忆、文本引导专家等统一为可验证的多模态证据，并通过验证集可靠性路由按数据集自适应选择、拒绝不可靠证据。在 MedicalExpert、archive、private 三个数据集上，**以 CLIP 基线（CLIP 架构 + 生物医学视觉塔的 BiomedCLIP 单分支）为统一起点**，本文完整框架将 Accuracy 分别提升 **+6.25% / +4.71% / +8.25%**，在 8 类主流骨干中三项指标均最优，且在 archive 与真实临床 private 上相对最强基线统计显著（McNemar p<0.05/0.01）；三个数据集的序数误差（MAE）一致下降，早期骨关节炎（KL1）召回明显提升。

**关键词**：膝关节骨关节炎；Kellgren–Lawrence 分级；序数分级；医学视觉–语言；多模态证据路由；双分支融合

## 1. 引言

膝关节骨关节炎（OA）是最常见的退行性关节疾病之一，KL 分级是其影像学严重度评估的国际标准。KL 分级具有两个常被忽视的本质特征：

1. **强医学语义**：每个等级由明确的影像学子结构定义——边缘骨赘（osteophyte）、关节间隙狭窄（joint space narrowing, JSN）、软骨下骨硬化（sclerosis）与骨变形（deformity）；
2. **序数结构**：KL0→KL4 是连续病程，相邻等级混淆的代价远小于跨级误判。

绝大多数已有工作以通用图像分类范式处理 KL 分级，将五个等级视为彼此独立的离散类别，既没有把分级背后的医学文本知识引入决策，也没有显式约束模型尊重等级之间的序数关系。与此同时，医学视觉–语言模型（如 BiomedCLIP）提供了天然的文本语义先验，但如何在不破坏已收敛视觉判别边界的前提下、稳定可靠地利用这种跨模态语义，仍是一个开放问题：直接把文本作为额外监督强行注入主干，往往在强视觉模型上产生负迁移，且在不同数据集间表现不一致。

本文从"**将文本知识转化为可验证、可选择的多模态医学证据**"这一视角出发，在 CLIP 视觉–语言架构上构建了一个文本–视觉多模态序数分级框架，提出**三项核心创新**——前两项构成单模型内的视觉强基线，第三项是文本–视觉多模态证据路由：

- **创新一：全局–局部跨注意力双分支编码。** 设计双向跨注意力 + 逐样本自适应门控的 `DualBranchCrossAttnFusion`，在单一模型内端到端融合 BiomedCLIP（ViT）的全局解剖建模与 ConvNeXt-V2 的局部骨赘细节，使两路在特征层互相交换信息，优于无特征交互的后集成。
- **创新二：多尺度局部建模与分层热启动优化。** 同时提取 ConvNeXt stage-2（细粒度骨赘纹理）与 stage-3（语义关节间隙）特征并以逐样本 scale gate 自适应融合（含 scale gate **偏置初始化技巧**，解决多尺度早期收敛失稳）；并以 Visual-Only / Backbone-Only **分层热启动 + 分支差异化学习率**解决引入新融合模块时的特征分布冲突，保护医学预训练特征（热启动作为该创新点的关键训练机制，不单列）。
- **创新三：RSOMR——可靠性感知语义序数多模态证据路由。** 将文本拆解为类别语义原型、序数病例记忆、文本引导专家等多模态证据，通过验证集可靠性路由按数据集自适应选择最可信证据并保留拒绝路径；文本不再是固定监督项，而是可被接受或拒绝的医学证据。

以 CLIP 基线为统一起点，三个数据集（公开 MedicalExpert、公开 archive、真实临床 private）上的系统实验表明：本文框架相对主流骨干与 CLIP 基线均取得一致、可验证的提升（AR/PV 相对最强基线统计显著），三数据集 MAE 一致下降、早期 OA 检出明显提升。

## 2. 相关工作

**KL 分级的深度学习方法。** 已有工作多基于 CNN（ResNet、DenseNet、EfficientNet、ConvNeXt）或 Transformer（ViT、PVT）骨干，将 KL 分级当作扁平五分类，少有显式利用医学文本语义或序数结构。

**医学视觉–语言模型。** BiomedCLIP 等在大规模生物医学图文对上预训练，提供了可迁移的跨模态语义。如何将其文本先验稳定迁移到下游细粒度分级、而不破坏视觉判别边界，是本文关注的核心。

**序数回归。** CORAL、CORN、cumulative-link 等显式建模序数结构。本文将序数语义注入**文本–视觉决策层**：RSOMR 以序数病例记忆与等级演进证据在决策层利用 KL 的序数结构，序数性同时由训练时序数损失承担。

## 3. 方法

### 3.1 总体框架

给定一张膝关节 X 光图像，框架分两层处理：**视觉强基线层**（创新一、二）输出融合特征 `fuse_feat` 与基础分类 logits `logits_base`；**文本–视觉多模态证据层**（创新三 RSOMR）在此之上注入可按数据集自适应选择/拒绝的多模态证据。整体结构见图 0。

![整体架构](figures/fig_architecture_full.png)
**图 0. 整体架构**：自顶向下展示三项核心创新——创新一（全局–局部跨注意力双分支融合）、创新二（多尺度局部建模与分层热启动）构成视觉强基线；创新三（RSOMR 可靠性感知多模态证据路由）在基座之上按数据集自适应选择/拒绝文本–视觉证据，输出 KL 序数等级。OCM-CLIP 基座内的轻量文本簇融合经消融验证非必要，不作为创新点。

#### 3.1.1 架构数据流与创新点位置（逐层详解）

整个框架自顶向下分为**视觉强基线层**与**文本–视觉多模态证据层**，共**三项核心创新**。设计主线：**先用视觉强基线把"看得准"做到位（创新一/二），再用 RSOMR 把"判得稳、有据"补齐（创新三）**。下面按图 0 的数据流逐级说明。

**输入层。** 单张膝关节正位 X 光 `I ∈ ℝ^{3×224×224}`（CLAHE 增强）。

**① 视觉强基线层**

- **双流编码（图第二行，左蓝右橙）**
  - 左：**全局分支 ViT（BiomedCLIP ViT-B/16）** → 512 维 `img_feat`，建模关节整体形态/对称/对线。
  - 右：**局部分支 ConvNeXt-V2 Large** → 同取 **stage-2（768d 细粒度骨赘）** 与 **stage-3（1536d 语义间隙）**，逐样本 scale-gate 融合为 `cvx_feat`。
  - 【**创新二：多尺度局部建模与分层热启动**】位于**右侧局部分支内部 + 整层训练过程**。scale-gate bias 初始化 `[-4,4]`（初始≈[0.004,0.996]）使早期等价单尺度、渐进引入细节；**分层热启动（Visual-Only/Backbone-Only）+ 分支差异化学习率**解决引入新融合模块时的特征分布冲突（图左红标，作为该创新点的关键训练机制）。
- **双分支跨注意力融合（图第三行，紫框）**
  - `img_feat` 与 `cvx_feat` 做**双向跨注意力**（global←local 定位骨赘、local←global 抑制非关节结构）+ 逐样本门控 → 融合图像特征。
  - 【**创新一：全局–局部跨注意力双分支融合**】位于**紫色融合框**——单模型替代后集成的核心，仅增 1.6M 参数。
- **基座出口（图第四行，绿框）**：经 OCM-CLIP 基座融合得 `fuse_feat` + 基础分类 `logits_base`，作为多模态层的统一接入点。
  - 说明：基座内部含一处继承自 OCM-CLIP 的轻量"簇感知文本原型融合"；经消融验证其对最终精度**非必要**（ME Δ=0、AR +0.3%噪声、PV −0.6%，见 §4.3），故**不作为本文创新点**，仅作基座实现细节，图中不予高亮。

**② 文本–视觉多模态证据层（创新三 RSOMR，本文核心多模态贡献）**

【**创新三：RSOMR 可靠性感知语义序数多模态证据路由**】（图第五行）—— 在 `fuse_feat / logits_base` 之上注入**可被选择/拒绝**的文本–视觉证据，不替换视觉判别边界。其结构为"**证据池 + 可靠性路由器**"：
- 证据池（候选）：**类别感知语义原型**、**文本引导序数病例记忆**、**文本引导专家**，外加序数过渡边界等候选；
- **可靠性路由器**：在验证集上**按数据集自适应选择**最可信证据、对不可靠证据置 `w=0` 拒绝。实测三数据集各自只激活一个主导证据（ME 文本专家 / AR 类别原型 / PV 病例记忆），序数过渡边界等候选被路由器拒绝——这正体现"拒绝不可靠证据"的设计（§4.4）。

**融合与输出（图第六行）。** 路由证据与 `logits_base` 合并 → 输出最终 **KL 序数 logits（KL0–KL4）**。

**训练损失（图底部）。** 基座训练损失 = 分类 CE + 序数损失（+ 原型/簇 CE）；**RSOMR 为推理层路由，无训练损失**。

**一句话设计思路**：视觉基线层用"全局–局部跨注意力（创新一）+ 多尺度与分层热启动（创新二）"把单模型判别力推到甚至超过后集成上界；再用 RSOMR（创新三）把文本知识转化为**可选择、可拒绝**的多模态医学证据，在不破坏视觉边界的前提下稳定增益。

### 3.2 创新一：全局–局部跨注意力双分支编码

**动机。** KL 分级同时依赖全局解剖对称（关节整体形态、对线，ViT 擅长）与局部病理细节（边缘小骨赘、间隙纹理，卷积擅长）。简单拼接或平均两路特征等价于后集成——两路在决策前**没有特征层交互**，无法互相"知晓"对方的发现。本文主张让全局路与局部路在特征层显式互相提问。

**做法。** `DualBranchCrossAttnFusion` 以**双向跨注意力 + 逐样本自适应门控**融合两路：

```
ViT 分支:      BiomedCLIP ViT-B/16        → 512-d 全局特征 img_feat
ConvNeXt 分支: ConvNeXt-V2 Large → 1536-d → Linear(1536→512) → 局部特征 cvx_feat

双向跨注意力:
  global←local:  Q=img_feat, K=V=cvx_feat → delta_g   (全局路查询局部以确认骨赘位置)
  local←global:  Q=cvx_feat, K=V=img_feat → delta_l   (局部路在全局对称背景下抑制非关节结构)

逐样本自适应门控:
  gate  = σ(Linear[img_feat ; cvx_feat]) ∈ [0,1]^512
  fuse  = gate ⊙ (img_feat + delta_g) + (1−gate) ⊙ (cvx_feat + delta_l) → LayerNorm → 分类头
```

两路梯度在端到端训练中互相影响，最终表示携带两种归纳偏置**协同后**的信息——这也是它优于后集成的原因（§5 分析）。

### 3.3 创新二：多尺度局部建模

**动机。** ConvNeXt-V2 Large 的标准出口是 stage-3（1536-d，对应语义级关节间隙），但早期小骨赘（KL1–KL2 的关键判别线索）在 stage-2（768-d，细粒度纹理）中更丰富，易被深层语义特征过度压缩。

**做法。** 从两个阶段同时取特征，经逐样本 scale gate 自适应融合：

```
stage-2 → Linear(768→512)  → f₂   (细粒度骨赘形态)
stage-3 → Linear(1536→512) → f₃   (语义关节间隙)
gates  = Softmax(Linear₂(GELU(Linear₁[f₂ ; f₃]))) ∈ [0,1]²
fused_local = gates[:,0]·f₂ + gates[:,1]·f₃ → Linear → LayerNorm → L2-Norm
```

**关键训练技巧——scale gate 偏置初始化。** scale gate 末层 bias 初始化为 `[-4, 4]`，使训练初期 gate≈[0.004, 0.996]、几乎完全依赖 stage-3（行为等价单尺度），随训练推进再逐渐学习何时利用 stage-2 细粒度特征。若不做此初始化，随机门控早期会产生噪声混合信号、干扰跨分支融合学习（最优 epoch 提前至 7–9 且性能低于单尺度）；加入后最优 epoch 延后至 15–18、性能反超单尺度。

### 3.4 创新二（训练机制）：分层热启动训练策略

**核心发现（一个隐性分布冲突）。** 旧 checkpoint 的 `fusion` 模块是在"纯 BiomedCLIP 特征"上训练的；引入 `cross_branch_fusion` 后 `img_feat` 分布已改变，若**全量加载**旧权重会产生负迁移（ME 上退回 0.8214，等同基线）。这一发现是双分支/多尺度可训练性的关键。

**两种热启动策略（按数据集规模选择）。**

- **策略 A — Visual-Only 热启动**（小数据集 ME/PV）：只加载 `visual.*`（151 个键），`fusion`/`classifier` 随新特征分布从零学起；`lr_visual=1e-5`（温和微调 ViT 以保护医学预训练）、`lr_convnext=1e-4`（充分训练 ConvNeXt）、`lr_fusion=3e-5`。
- **策略 B — Backbone-Only 热启动**（单尺度→多尺度迁移，大数据集 AR）：从单尺度最优 checkpoint 仅迁移 `visual.* + visual_local.backbone.*`（并映射 `proj→proj_s3`），**跳过 fusion/classifier**——因为单尺度下融合输入是单一局部特征、多尺度下是 stage-2/3 门控融合特征，分布不同，直接加载不如从新分布重学；AR 用极小 lr（`lr_visual=3e-6`、`lr_convnext=1e-6`、`lr_fusion=1e-4`）近乎冻结已充分适配的骨干，只学融合。

**为何重要。** 该策略将"在强预训练骨干上引入新融合模块"这一普遍难题，转化为可控的分层迁移问题，是双分支与多尺度两项创新能够稳定生效的前提。

> 视觉基线小结：创新一、二配合，使**单一模型**在 MedicalExpert/private 上超过后集成上界、在 archive 上接近上界，为多模态证据提供稳定的视觉基座（详见 §4.3 Table 2 / Table 2B）。

### 3.5 创新三：RSOMR——可靠性感知语义序数多模态证据路由

**动机。** 强视觉基线之上，直接文本监督易破坏已收敛判别边界，且同一文本约束跨数据集可靠性不一。本文主张：**文本不应是固定监督项，而应是可验证、可选择的多模态医学证据。**

**四类多模态证据。**

1. **类别感知语义原型证据**：以 KL 文本原型与训练集视觉原型共建类别语义锚点，支持 class-wise 权重，使文本仅在可靠类别边界介入；
2. **文本引导序数病例记忆证据**：以 KL 文本标签图结合等级邻接关系，对相似病例 cache 证据进行序数感知的文本图传播；
3. **文本序数边界证据**：以相邻等级 transition text 建模病程演进边界，将图像特征与"是否跨过某 KL 边界"对齐；
4. **文本引导专家证据**：将文本引导训练得到的 checkpoint 视为多模态专家，路由其与视觉基线的互补部分，而非要求其单独超越视觉基线。

**可靠性路由。** 设视觉基线输出 `logits_base`，各证据源输出 `logits_{proto,cache,boundary,expert}`，在验证集上选择权重：

```
logits_final = norm(logits_base)
             + w_proto    · norm(logits_proto)
             + w_cache    · norm(logits_cache)
             + w_boundary · norm(logits_boundary)
             + w_expert   · norm(logits_expert)
```

关键在于保留 `w=0` 的退化路径：当某证据源在验证集上不可靠时路由自动拒绝它。由此，文本知识被结构化为可按数据集自适应选择/拒绝的多模态证据——这正是 RSOMR 区别于普通 prompt ensemble 或 logits ensemble 的多模态本质。其序数性来自序数病例记忆与等级演进证据，并与训练时序数损失互补。

| ViT 全局塔（BiomedCLIP） | 86.2 | 视觉骨干 |
| ConvNeXt-V2 Large 局部塔 | 198.4 | 视觉骨干 |
| 跨注意力双分支融合（创新一） | **1.6** | 引入特征层交互 |
| 多尺度门控 + 融合（创新二） | 2.1 | stage-2/3 自适应 |
| 簇感知融合 + 分类头 | 0.3 | OCM-CLIP 基座 |
| **推理总计（视觉）** | **≈288** | 单一前向 |
| RSOMR 多模态路由（创新三） | **0（推理时）** | 验证集选标量权重，无新增可训练参数 |
| 文本编码器（PubMedBERT，离线） | ~110 | 仅离线预计算原型，推理不参与 |

**要点：** 双分支跨注意力**仅以 1.6M 参数**（占视觉总量 0.6%）引入了后集成所缺失的特征层交互；RSOMR 在推理时为**零新增可训练参数**（仅验证集选出的标量权重）；文本编码器仅离线预计算 KL 文本原型，推理阶段不加载。因此本文以与"运行两个视觉骨干"相当的代价，换来单前向、特征级交互、可路由多模态证据的完整能力，而后集成需维护两套独立模型、两次前向且两路无交互。

---

## 5. 分析与讨论

**为何双分支优于后集成。** 后集成两路决策前无特征层交互；双分支跨注意力使全局路"查询"局部以确认骨赘位置、局部路在全局对称背景下抑制非关节结构，端到端联合优化使表征携带两种归纳偏置协同后的信息。

**多模态证据为何跨数据集稳定。** 不同数据集的可靠文本证据并不相同：ME 视觉基线已强、需文本专家的互补边界；AR 需类别级文本语义锚定早期边界；PV 真实队列更依赖样本级病例记忆与序数标签图。RSOMR 不假设某种文本监督普遍有效，而是把文本拆成多层多模态证据、由验证集选择可靠来源——这是其在三数据集同步获益的根本原因。

**序数性的处理。** KL 的序数结构由两处协同承担：训练时的序数损失约束基座输出，RSOMR 在决策层引入序数病例记忆与等级演进证据。序数回归基线对比（§4.5）也表明，在同一特征上序数建模相对 nominal 一致改善 QWK，印证该设计取向。

**适用性。** 框架在视觉因数据受限而不充分时（如真实临床 private 集）多模态增益最大；在视觉已接近饱和的基准上，多模态证据通过可靠性路由保持稳定、并以序数与可解释性收益为主。这一"按可靠性自适应"的设计使其在异构临床数据上具备良好的稳健性与可迁移性。

---

## 6. 结论

本文提出一个面向膝关节 KL 分级的文本–视觉多模态序数框架，含三项核心创新：①全局–局部跨注意力双分支融合与②多尺度局部建模与分层热启动，在单一模型内构成强视觉基线（在 ME/PV 上超过后集成上界）；③RSOMR 可靠性感知语义序数多模态证据路由，将 KL 类别原型、序数病例记忆与文本引导专家统一为可验证、可选择/拒绝的多模态证据。在三个数据集上，本文方法相对主流骨干与 CLIP 基线一致提升，AR/PV 相对最强基线统计显著，三数据集 MAE 一致下降、早期 OA 召回大幅改善。该框架表明：在强视觉基线之上，文本知识的价值不在于作为固定监督项，而在于被结构化为按可靠性自适应选择的多模态医学证据。
