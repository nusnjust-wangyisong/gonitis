# 复现说明

本文件说明如何在本地复现论文的三项核心创新与全部结果。最终模型与报告见 `docs/report3.md`。

最终指标（测试集 Accuracy / Macro-F1 / MAE）：

| 数据集 | Accuracy | Macro-F1 | MAE |
|--------|----------|----------|-----|
| MedicalExpert (ME) | 0.8839 | 0.8821 | 0.1473 |
| archive (AR) | 0.7252 | 0.7346 | 0.3086 |
| private (PV) | 0.7064 | 0.7012 | 0.3333 |

三项创新：①全局–局部跨注意力双分支融合；②多尺度局部建模与分层热启动；③RSOMR 可靠性感知语义序数多模态证据路由。最终模型 = 视觉强基线（①②）+ RSOMR 路由（③）。

## 2. 环境

- OS：Linux；Python 3.9–3.10；NVIDIA GPU（显存 ≥ 16GB 用于训练，推理 ≥ 8GB）。
- 关键依赖：PyTorch（CUDA 版）、open_clip_torch、timm、transformers、scikit-learn、opencv-python、matplotlib、scipy。

conda 路径：

```bash
conda create -n koa python=3.10 -y
conda activate koa
pip install -r requirements.txt
```

venv + pip 备选：

```bash
python -m venv .venv && source .venv/bin/activate
```

> 说明：复现最终指标只需各目录下的 `best_model.pth`（约 1.6GB/个），无需完整 checkpoint 目录（含优化器状态，约 5GB/个）。

## 5. 复现步骤

### 5.1 训练视觉强基线（创新一 + 创新二）

ME（Visual-Only 热启动）：

```bash
python main.py --variant full_dual --backbone biomedclip --convnext_multi_scale \
  --data_root ../公开数据集/MedicalExpert-split \
  --init_checkpoint experiments/checkpoints/checkpoints_biomedclip_bm224/MedicalExpert-split_full/best_model.pth \
  --init_visual_only --lr_visual 1e-5 --lr_convnext 1e-4 --lr_fusion 3e-5 \
  --batch_size 32 --epochs 60
```

AR / PV（Backbone-Only 热启动，从单尺度 checkpoint 迁移）见 `docs/report2.md` §7 的完整命令。

### 5.2 RSOMR 多模态证据路由（创新三，推理层，无需训练）

```bash
# ME：文本引导专家路由
python scripts/semantic_ordinal_transport_adapter_eval.py --dataset me --objective accuracy \
  --expert_step 0.02 \
  --extra_checkpoints \
    experiments/checkpoints/checkpoints_universal_text_me_gtp/MedicalExpert-split_full_dual/best_model.pth \
    experiments/checkpoints/checkpoints_text_curriculum_me_lt005_stage2/MedicalExpert-split_full_dual/best_model.pth \
    experiments/checkpoints/checkpoints_text_curriculum_me_stage2/MedicalExpert-split_full_dual/best_model.pth \
  --extra_names gtp curriculum005 curriculum02 \
  --out experiments/results/semantic_ordinal_transport_adapter_fine

# AR：类别感知语义原型
python scripts/text_visual_proto_blend_eval.py --dataset ar --objective macro_f1 --classwise \
  --out experiments/results/text_visual_proto_blend_aligned_classwise

# PV：文本引导序数病例记忆
python scripts/semantic_ordinal_transport_adapter_eval.py --dataset pv --objective macro_f1 \
  --out experiments/results/semantic_ordinal_transport_adapter
```

## 8. 随机性与误差

- 推理与 RSOMR 路由为确定性，给定相同权重应可复现表中数字。
- 重新训练涉及随机性（数据增强、初始化、CUDA 非确定算子），单次结果可能有约 ±0.5–1.0 个百分点波动；private 临床集上的多模态增益经 10 种子验证统计显著。
- 不承诺逐位一致。

## 9. 常见问题

- **PubMedBERT / 预训练权重下载失败**：检查网络或代理；可手动下载 HuggingFace 上的 `microsoft/BiomedNLP-...` 与 BiomedCLIP 放到对应缓存目录。
- **显存不足**：减小 `--batch_size`；推理可用单卡 8GB。
- **找不到 checkpoint**：确认 §4 的 `best_model.pth` 已放到对应目录，路径与 `scripts/semantic_ordinal_transport_adapter_eval.py` 中 `CFG` 一致。
- **数据集路径不对**：数据集默认在仓库父目录的 `../公开数据集`、`../私有数据集`；可在各脚本 `CFG` 或 `--data_root` 中调整。
- **Spatial-Mamba 报错**：确认官方代码与 Tiny 权重已就位（§6），否则跳过该对比模型。
